TODOリスト
基本仕様：
・TODOの登録
　登録内容はタイトル、詳細、締切日を入れるようにした。
　詳細、締切日は任意で登録できるようにした。
　登録時に登録日と完了チェックの状態も自動的に追加して保存している。
・TODOリストの表示
　登録内容を表示する。1つの内容を2段構成で表示する。
　上の行に完了チェック、タイトル、締切、登録日があり、
　下の行に詳細事項がある。
　詳細事項は長くなる場合があるため折り畳みが出来るようにしてある。
・TODOの完了
　TODOリストにあるチェックボタンをクリックすることで完了させることが出来る。
　完了したためこのリストの背景は灰色に変化する。
　なお、誤って完了したとチェックする可能性があるため、
　確認とチェックを外せる機能がある。

内部仕様（データ構造）：
・ローカルストレージのデータ構造
　ローカルストレージのkeyは接頭辞に「todo」を付け、
　その後ろにDate().getTime()で現在時刻を秒で表す関数を用いて
　ユニークなキーとしている（同時刻に登録できないことを前提とする）。
　valueはJSON形式を文字列化して保存している。詳細は次項へ。
・JSONの構造（TODOリストのデータ構造）
　以下のような連想配列で保存している。
　| key     | type   | 内容           |
 =====================================
　| isCheck | bool   | チェックされているか |
　| created | Date   | 登録日         |
　| deadline| Date   | 締め切り日      |
　| title   | string | タイトル         |
　| detail  | string | 詳細           |
　=====================================
　この連想配列をJSON形式に文字列化してローカルストレージのvalueとして保存している。

追加機能：
・締切日の登録が可能
　必ず必要ではないため、指定がなければ「指定なし」と表示するようにした
・詳細の登録を可能にした
　TODOは簡単に記述するものだが、中には詳細を書きたいことがあるため、
　そこに書けるようにした。
　また、長い文章になるとリストとして見づらくなるため、プルダウンで詳細を見れるようにした。
・完了項目の背景を灰色にした
　完了項目は目立たせる必要はないため灰色にした
